# specs/ — 設計指示書フォルダ

## このフォルダについて

Claude AI（ブレスト・設計担当）が作成した実装指示書を管理するフォルダです。
Claude Code（実装担当）は、実装前にこのフォルダの関連ファイルを読んでから作業してください。

## ファイル命名規則

```
XX_機能名.md
```

- `XX`：連番（01, 02, 03...）。実装の優先順に振る
- `機能名`：何の指示書かがわかる短い名前（英語）

## 現在の指示書一覧

| 番号 | ファイル名 | 内容 | 状態 |
|------|-----------|------|------|
| 00 | `00_project_overview.md` | プロジェクト全体の設計方針・役割分担・開発フロー | — 常時参照 |
| 01 | `01_auto_verification.md` | 自動検証＋自動承認（第1弾） | ✅ 実装完了 |
| 02 | （未作成） | アラート・監視システム（第2弾） | 🔜 次回予定 |

## 状態の凡例

- ✅ 実装完了
- 🔧 実装中
- 🔜 次回予定
- 📝 設計中（ブレスト段階）
- ❌ 取り消し

## Claude Codeへの注意事項

- 新しい機能の実装を始める前に、必ず該当の指示書を読むこと
- 過去の指示書も目を通すこと（設計の背景や判断理由が書いてある）
- 指示書にない判断が必要な場合は、実装を進める前に確認すること
- 実装完了後は、完了報告の要点をこのREADMEの一覧表に反映すること
